# 프롬프트 엔지니어링: 건축가의 작업 지시서

## 1. 아키텍트의 시선: 개떡 vs 찰떡

"개떡같이 말해도 찰떡같이 알아들어라"는 사람끼리나 통하는 말입니다.
AI(건축가)에게 개떡같이 말하면 **개떡 같은 건물**을 지어줍니다.

프롬프트 엔지니어링은 **"AI가 딴짓을 못 하게 가드레일을 치는 기술"**입니다.
아키텍트라면 단순한 질문이 아니라 **"완벽한 설계도(Prompt)"**를 던져야 합니다.

## 2. 프롬프트의 5대 요소 (The Structure)

좋은 프롬프트는 다음 5가지 요소를 반드시 포함합니다. (비유: **P.C.T.O.F**)

1.  **Persona (페르소나)**: 너는 누구냐? (예: "너는 20년 차 시니어 개발자야.")
2.  **Context (맥락)**: 상황이 어때? (예: "초보 개발자에게 파이썬을 가르치고 있어.")
3.  **Task (임무)**: 뭘 해야 해? (예: "리스트 개념을 설명해줘.")
4.  **Constraints (제약)**: 하지 말아야 할 것은? (예: "전문 용어 쓰지 말고, 비유를 들어서 설명해.")
5.  **Format (형식)**: 결과물은 어떻게? (예: "마크다운 표 형식으로 정리해줘.")

### ❌ 나쁜 예 (The Bad)
> "리스트 설명해줘."
> (결과: 교과서적이고 지루한 설명이 나옴)

### ✅ 좋은 예 (The Good)
> **[System Prompt]**
> 당신은 친절한 '코딩 유치원 선생님'입니다. (Persona)
> 코딩을 처음 배우는 7살 아이에게 설명한다고 가정하세요. (Context)
>
> **[User Prompt]**
> 파이썬의 '리스트' 개념을 '장난감 바구니'에 비유해서 설명해주세요. (Task & Constraints)
> 설명 끝에는 퀴즈를 하나 내주세요. (Format)

## 3. 핵심 기술 (Core Techniques)

### 3.1 Few-shot Prompting (예시 보여주기)
백 마디 설명보다 한 번의 예시가 낫습니다.

*   **Zero-shot (예시 없음)**: "이거 감정 분석해."
*   **Few-shot (예시 있음)**:
    > "리뷰: 이 영화 진짜 최악. -> 부정"
    > "리뷰: 시간 가는 줄 몰랐음. -> 긍정"
    > "리뷰: 배우 연기는 좋은데 스토리가 엉망. -> 중립"
    >
    > "리뷰: 돈 아까워 죽는 줄. -> ?"

### 3.2 Chain of Thought (CoT, 단계별로 생각하기)
AI에게 **"생각할 시간"**을 주는 것입니다. 수학 문제나 논리 문제에서 필수입니다.

*   **Bad**: "5 + 3 * 2는?"
*   **Good**: "**단계별로 생각해서 풀어봐(Let's think step by step).** 먼저 곱셈을 하고, 그 다음에 덧셈을 해."

## 4. 코드 설계도: 프롬프트 템플릿

매번 긴 프롬프트를 칠 수 없으니, **구멍 뚫린 템플릿**을 만들어 씁니다.

```python
template = """
당신은 {role}입니다.
사용자의 질문: {question}

다음 규칙을 지켜서 답하세요:
1. {tone} 말투로 대답할 것.
2. 3줄 요약을 포함할 것.
"""

# 구멍 채우기
prompt = template.format(
    role="건축 아키텍트",
    question="AI의 미래는?",
    tone="전문적이지만 희망적인"
)

print(prompt) # 완성된 지시서 확인
```
