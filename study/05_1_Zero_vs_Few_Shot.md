# 🦁 Zero-shot vs Few-shot: AI 가르치기

안녕하세요, 아키텍트님!
이 두 가지는 **"AI에게 업무를 지시하는 스타일"**의 차이입니다.
가장 확실하게 이해시켜 드리겠습니다.

---

## 1. 개념 비교 (The Concept)

### 🥚 Zero-shot (제로 샷)
*   **의미**: "받아 적어." (예시 0개)
*   **비유**: **"경력직 사원"**에게 일 시키기.
    *   "이거 번역해."
    *   "이거 요약해."
    *   "알아서 잘하겠지" 하고 믿고 맡기는 방식입니다.
*   **장점**: 빠르고 간편합니다.
*   **단점**: 내 의도와 다르게 결과가 나올 수 있습니다. (형식, 말투 등)

### 🐣 Few-shot (퓨 샷)
*   **의미**: "이것 좀 봐. 이렇게 하는 거야." (예시 3~4개)
*   **비유**: **"신입 사원"**에게 일 시키기.
    *   "자, 봐봐. A가 들어오면 B를 내놓는 거야. 알겠지? 이제 이거 해봐."
    *   패턴을 보여줘서 그대로 따라 하게 만드는 방식입니다.
*   **장점**: **정확도**가 비약적으로 상승합니다. 특히 **형식(Format)**을 맞출 때 강력합니다.
*   **단점**: 프롬프트가 길어져서 돈(토큰)이 좀 더 듭니다.

## 2. 실전 예시: "감성 분석기" 만들기

사용자의 리뷰를 보고 긍정(Positive)인지 부정(Negative)인지 맞추는 AI를 만든다고 칩시다.

### ❌ Zero-shot 접근 (불안정함)
그냥 "이거 분석해"라고 던집니다.

```python
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": "너는 감성 분석기야."},
        {"role": "user", "content": "이 영화 진짜 시간 아깝다."}
    ]
)
# 결과 예상: "이 문장은 부정적인 감정을 담고 있습니다." (너무 길다!)
# 나는 딱 "부정" 두 글자만 원하는데!
```

### ✅ Few-shot 접근 (완벽함)
"내가 원하는 답안지 스타일"을 미리 보여줍니다.

```python
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "system", "content": "너는 감성 분석기야. 무조건 '긍정' 또는 '부정'으로만 대답해."},
        
        # [예시 1]
        {"role": "user", "content": "와, 진짜 인생 영화였다."},
        {"role": "assistant", "content": "긍정"}, # AI가 이렇게 대답했었다고 '가짜 기억'을 심어줌
        
        # [예시 2]
        {"role": "user", "content": "돈 아까워 죽는 줄."},
        {"role": "assistant", "content": "부정"},
        
        # [예시 3]
        {"role": "user", "content": "배우 연기는 좋은데 스토리가 산으로 가네."},
        {"role": "assistant", "content": "부정"}, # 헷갈리는 것도 예시로 정해줌
        
        # [실전 문제]
        {"role": "user", "content": "이 영화 진짜 시간 아깝다."}
    ]
)
# 결과: "부정" (딱 원하는 대로 나옴!)
```

## 3. 언제 뭘 써야 할까?

| 상황 | 추천 방식 | 이유 |
| :--- | :--- | :--- |
| **일반적인 질문** | **Zero-shot** | "미국 대통령이 누구야?", "파이썬 설명해줘" 같은 건 예시가 필요 없음. |
| **특이한 말투** | **Few-shot** | "무조건 '멍!'으로 끝내." -> 예시를 몇 개 보여줘야 잘 따라 함. |
| **JSON/코드 변환** | **Few-shot** | "입력된 텍스트를 `{ 'name': ... }` 형태로 바꿔." -> 예시 없으면 엉망으로 줌. |
| **분류 (Classification)** | **Few-shot** | 애매한 기준을 예시로 잡아줘야 함. |


## 4. 자주 하는 오해 (FAQ) 🚨

> **Q: "첫 질문은 Zero-shot이고, 대화를 계속하면 그게 Few-shot이 되는 건가요?"**

**아닙니다!** 이 둘은 목적이 다릅니다.

1.  **대화의 역사 (Chat History)**:
    *   **목적**: "문맥 유지" (아까 했던 말 기억하기)
    *   **내용**: 사용자와 AI가 실제로 주고받은 *진짜 대화*.
    *   **결과**: 대화가 길어질수록 자연스럽게 문맥이 쌓임.

2.  **퓨샷 프롬프팅 (Few-shot Prompting)**:
    *   **목적**: "지능 향상 / 교육" (패턴 가르치기)
    *   **내용**: 개발자가 일부러 지어낸 *가짜 대화(Fake History)*.
    *   **시점**: **첫 질문을 던지기 전**에 이미 `system`과 `user` 메시지 사이에 끼워 넣는 것.

즉, **Few-shot은 "첫 질문부터" 잘하게 만들기 위해 미리 족보를 보여주는 기술**입니다!
