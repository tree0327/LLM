# RAG 개념 재정리: "컨닝 페이퍼" 📝

아키텍트님, 어렵게 생각하지 마세요!
RAG의 본질은 **"정답을 찾아서 프롬프트에 몰래 붙여넣기"**입니다.

---

## 1. 3줄 요약 (The Essence)

1.  **검색 (Retrieve)**: 질문과 관련된 문서를 찾는다. (구글링)
2.  **참조 (Augment)**: 찾은 내용을 프롬프트에 **복사+붙여넣기** 한다.
3.  **생성 (Generate)**: "위 내용을 보고 대답해"라고 시킨다.

이게 끝입니다. 진짜로요.

---

## 2. 왜 이게 필요한가요?

### 시나리오: "우리 회사 회식비 얼마야?"
*   **그냥 LLM**: (학습한 적 없음) "보통 회사들은 3만 원 정도 줍니다..." (헛소리/Hallucination)
*   **RAG 사용**:
    1.  시스템이 몰래 `회사내규.pdf`를 뒤져서 "회식비: 1인당 5만 원" 부분을 찾음.
    2.  시스템이 프롬프트를 이렇게 조작함:
        > "다음 내용을 참고해서 답해줘: **[회식비는 1인당 5만 원이다.]**
        > 질문: 회식비 얼마야?"
    3.  **LLM**: "5만 원입니다." (정답!)

즉, RAG는 **"LLM에게 족보(컨닝 페이퍼)를 쥐여주고 시험 보게 하는 기술"**입니다.
복잡한 `Embedding`, `Vector DB` 같은 건 전부 **"그 족보를 빨리 찾기 위한 도구"**일 뿐입니다.

---

## 3. 코드 흐름 다시 보기

```python
# 1. 족보 찾기 (Retrieve)
# 질문("회식비 얼마?")과 비슷한 문서를 DB에서 찾아옴
docs = vector_db.similarity_search("회식비 얼마?")
# doc 내용: "제7조: 회식비는 5만원..."

# 2. 족보 쥐여주기 (Prompting)
# 찾은 docs 내용을 프롬프트에 끼워 넣음 (이게 핵심!)
final_prompt = f"""
참고 문서: {docs[0].page_content}

질문: 회식비 얼마야?
"""

# 3. 답변하기 (Generate)
llm.predict(final_prompt)
```

**결론:**
RAG는 거창한 인공지능 기술이라기보다, **"검색해서 붙여넣기(Search & Paste)"**를 자동화한 시스템입니다.
