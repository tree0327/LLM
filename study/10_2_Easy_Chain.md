# 체인(Chain) 아주 쉽게 이해하기: "이어달리기" 🏃

아키텍트님, 죄송합니다! 제가 잠시 욕심을 부려 어려운 용어를 썼네요.
다 잊고, 딱 하나만 기억합시다.

**Chain(체인)은 "이어달리기"입니다.**

---

## 1. 지금까지 우리가 한 방식 (수동)

지금까지는 아키텍트님이 **혼자서 1인 3역**을 했습니다.

1.  (프롬프트 작성자): "자, 이렇게 질문해야지..." (`prompt = ...`)
2.  (AI 호출자): "GPT야, 대답해봐." (`response = llm.invoke(...)`)
3.  (결과 정리자): "답변에서 글자만 뽑자." (`print(response.content)`)

**문제점**: 매번 변수에 담고, 꺼내고, 다시 넣고... 귀찮습니다.

## 2. 체인을 쓰는 방식 (자동)

이제 **선수 3명**을 고용해서 한 줄로 세웁니다.

*   **1번 주자 (Prompt)**: 질문을 받아서 이쁘게 포장함.
*   **2번 주자 (Model)**: 포장된 걸 받아서 답변함.
*   **3번 주자 (Parser)**: 답변을 깔끔하게 다듬음.

그리고 아키텍트님은 이렇게 명령만 내립니다.
> **"얘들아, 서로 어깨에 손 올려! ( 연결: `|` )"**

```python
# 1번 | 2번 | 3번
chain = prompt | model | parser
```

## 3. 왜 `|` 모양인가요?

이 기호(`|`)는 파이프(Pipe)라고 부릅니다.
수도관 파이프처럼 **"물이 흐르는 통로"**를 연결해준다고 생각하세요.

*   1번 주자가 만든 정보가 파이프를 타고 -> 2번에게 가고 -> 3번에게 갑니다.
*   우리는 중간 과정을 신경 쓸 필요 없이 **"수도꼭지(invoke)"**만 틀면 물(정답)이 콸콸 나옵니다.
